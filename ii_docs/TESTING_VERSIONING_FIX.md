# Инструкции по тестированию исправления версионирования

## Исправленная проблема

В функции `ГраницыУдаленияОбъектов()` модуля `ВерсионированиеОбъектов` была исправлена критическая логическая ошибка, из-за которой все версии объектов удалялись через месяц, независимо от настроенных сроков хранения (включая "Бессрочно").

## Что было исправлено

**Строки 2022-2033 в модуле `CommonModules/ВерсионированиеОбъектов/Ext/Module.bsl`**

### Было (НЕВЕРНО):
```bsl
НеверсионируемыеТипы = ОбщегоНазначенияКлиентСервер.РазностьМассивов(
    ВсеТипы, СписокТипов); //неверсионируемое то что не в результате запроса
```

Переменная `СписокТипов` содержала типы только из последней итерации цикла, а не все типы из результата запроса.

### Стало (ПРАВИЛЬНО):
```bsl
// Собираем все типы, для которых есть настройки версионирования
ВсеВерсионируемыеТипы = Новый Массив;
Для Каждого СтрокаГраницы Из Результат Цикл
    Для Каждого Тип Из СтрокаГраницы.СписокТипов Цикл
        ВсеВерсионируемыеТипы.Добавить(Тип);
    КонецЦикла;
КонецЦикла;

// Неверсионируемое - это то, что не в результате запроса
НеверсионируемыеТипы = ОбщегоНазначенияКлиентСервер.РазностьМассивов(
    ВсеТипы, ВсеВерсионируемыеТипы);
```

Теперь собираются ВСЕ типы из всех групп результата запроса.

### Шаг 3: Проверка результата

#### Ожидаемый результат:

1. **Типы с настройкой "Бессрочно"**:
   - `ГраницаУдаления = '000101010000'` (1 января 1 года)
   - Эти версии НЕ должны удаляться

2. **Типы с настройкой "За последний год"**:
   - `ГраницаУдаления = ТекущаяДатаСеанса() - 12 месяцев`
   - Версии старше года должны удаляться

3. **Типы с настройкой "За последние 6 месяцев"**:
   - `ГраницаУдаления = ТекущаяДатаСеанса() - 6 месяцев`
   - Версии старше 6 месяцев должны удаляться

4. **Типы с настройкой "За последние 3 месяца"**:
   - `ГраницаУдаления = ТекущаяДатаСеанса() - 3 месяца`
   - Версии старше 3 месяцев должны удаляться

5. **Типы с настройкой "За последний месяц"**:
   - `ГраницаУдаления = ТекущаяДатаСеанса() - 1 месяц`
   - Версии старше месяца должны удаляться

6. **Типы БЕЗ настроек** (неверсионируемые, например, версии по обмену данными):
   - `ГраницаУдаления = ТекущаяДатаСеанса() - 1 месяц`
   - Версии старше месяца должны удаляться

#### Как проверить в данных:

1. Откройте **Регистр сведений "НастройкиВерсионированияОбъектов"**
2. Проверьте, какие типы объектов имеют срок хранения "Бессрочно"
3. Откройте **Регистр сведений "ВерсииОбъектов"**
4. Отфильтруйте версии по этим типам объектов
5. Убедитесь, что старые версии (например, годовой давности) присутствуют в регистре
