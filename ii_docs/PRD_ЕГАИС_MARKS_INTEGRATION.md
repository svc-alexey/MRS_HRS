# PRD: Интеграция с марками ЕГАИС через Simphony FiscalDB API

**Дата:** 17 марта 2026  
**Версия:** 2.0  
**Приоритет:** High

---

## 1. Цель проекта

Адаптировать интеграцию 1С с Simphony для работы с акцизными марками ЕГАИС через новое API FiscalDB, заменив устаревшие методы работы со штрихкодами и марками.

### Ключевые результаты
- [ ] Успешная выгрузка марок ЕГАИС в фискальную базу Simphony
- [ ] Корректная загрузка проданных марок из чеков
- [ ] Отсутствие использования устаревших методов `/egais/barcodes`
- [ ] Транзакционная целостность при создании позиций с ЕГАИС-атрибутами

---

## 2. Изменения метаданных

### 2.1 План обмена `питУдаленныеКассы`

Добавить новые реквизиты:

| Реквизит | Тип | Назначение |
|----------|-----|------------|
| `MRS_HierStrucId` | Число(10) | ID структуры для создания кнопок в меню |
| `MRS_ParentRvcId` | Число(10) | ID родительской ветки для создания марок |
| `MRS_ИмяФискальнойБазы` | Строка(100) | Имя виртуального склада в FiscalDB |

**Заполнение:** При указании префикса (rvc_id) выполняется запрос к `/referencedata/rvc-hierarchy` и автоматически заполняются все три реквизита.

### 2.2 Справочник `питМеню`

Добавить новые реквизиты:

| Реквизит | Тип | Назначение |
|----------|-----|------------|
| `MRS_ЕдиницаИзмеренияФБ` | ПеречислениеСсылка | Единица измерения для FiscalDB |
| `MRS_ТипМаркиФБ` | ПеречислениеСсылка | Тип марки (бутылка/порция/кег) |

### 2.3 Новые перечисления

**Перечисление: `MRS_ЕдиницыИзмеренияФБ`**

| Значение | Код для API |
|----------|-------------|
| Штуки | 0 |
| Граммы | 10 |
| Килограммы | 11 |
| Миллилитры | 40 |
| Литры | 41 |

**Перечисление: `MRS_ТипыМарокФБ`**

| Значение | Код для API |
|----------|-------------|
| Бутылка | 1 |
| Порция | 2 |
| Коктейль | 3 |
| РазливноеПиво | 4 |
| Кег | 5 |

### 2.4 Документ "Чек ККМ"

Добавить табличную часть **`МаркиЕГАИС`**:

| Поле | Тип | Назначение |
|------|-----|------------|
| НомерПозицииЧека | Число(4) | Связь с позицией в ТЧ "Товары" (через ckpos) |
| КодМарки | Строка(150) | Код марки ЕГАИС |
| ОбъёмПродано | Число(15,3) | Проданный объём из марки (напрямую из `vol` API) |

**Важно:** Одна позиция чека может иметь несколько марок (при использовании нескольких бутылок для одной порции).

### 3.2 Создать новую функцию `ВыгрузитьМаркуВФискальнуюБазу()`

**Назначение:** Выгрузка одной марки ЕГАИС в фискальную базу Simphony.

**Параметры:**
- `Позиция` - СправочникСсылка.питМеню
- `КодМарки` - Строка - код марки ЕГАИС
- `ОстатокМарки` - Число - остаток марки в литрах
- `КассовыйУзел` - ПланОбменаСсылка

**Алгоритм:**

```bsl
Функция ВыгрузитьМаркуВФискальнуюБазу(Позиция, КодМарки, ОстатокМарки, КассовыйУзел)
    
    // 1. Получить идентификаторы из плана обмена
    ParentRvcId = КассовыйУзел.MRS_ParentRvcId;
    StoreName = КассовыйУзел.MRS_ИмяФискальнойБазы;
    
    // 2. Сформировать тело запроса
    ТелоЗапроса = Новый Структура;
    ТелоЗапроса.Вставить("objnum", Позиция.КодОбъектаSimphony);
    ТелоЗапроса.Вставить("code", КодМарки);
    ТелоЗапроса.Вставить("store", StoreName);
    ТелоЗапроса.Вставить("name", Позиция.Наименование);
    ТелоЗапроса.Вставить("ean", " ");  // Всегда пробел
    ТелоЗапроса.Вставить("info", Null);
    ТелоЗапроса.Вставить("groupid", 0);
    ТелоЗапроса.Вставить("tare", "БУТЫЛКА");
    ТелоЗапроса.Вставить("volume", ОстатокМарки);
    ТелоЗапроса.Вставить("remainder", ОстатокМарки);
    ТелоЗапроса.Вставить("state", "");  // Пустая строка
    ТелоЗапроса.Вставить("fsrarid", Null);
    ТелоЗапроса.Вставить("keeptime", 0);  // Для бутылок всегда 0
    ТелоЗапроса.Вставить("proddate", Null);
    ТелоЗапроса.Вставить("expdate", Null);
    ТелоЗапроса.Вставить("opendate", Null);
    ТелоЗапроса.Вставить("solddate", Null);
    
    // 3. Отправить запрос
    Endpoint = СтрШаблон("/rvc/%1/fiscaldb/marks", ParentRvcId);
    HTTPОтвет = ОтправитьЗапросAPI("POST", Endpoint, ТелоЗапроса, Заголовки);
    
    // 4. Обработать ответ
    Если HTTPОтвет.КодСостояния = 200 Тогда
        Возврат Истина;
    Иначе
        ЗаписьЖурналаРегистрации("Выгрузка марок Simphony",
            УровеньЖурналаРегистрации.Ошибка,,,
            СтрШаблон("Ошибка выгрузки марки %1: HTTP %2", КодМарки, HTTPОтвет.КодСостояния));
        Возврат Ложь;
    КонецЕсли;
    
КонецФункции
```

## 4. Модуль `MRS_МенеджерОбменаMicros`

### 4.1 Создать функцию `ПолучитьПроданныеМаркиHTTP()`

**Назначение:** Загрузка проданных марок из фискальной базы.

**Параметры:**
- `ПараметрыВыборки` - структура с параметрами (НачалоПериода, КонецПериода, НомерЧека, МассивТочекПродаж)

**Алгоритм:**

```bsl
Функция ПолучитьПроданныеМаркиHTTP(ПараметрыВыборки)
    
    ТаблицаМарок = Новый ТаблицаЗначений;
    ТаблицаМарок.Колонки.Добавить("id", Новый ОписаниеТипов("Число"));
    ТаблицаМарок.Колонки.Добавить("cknum", Новый ОписаниеТипов("Число"));
    ТаблицаМарок.Колонки.Добавить("ckpos", Новый ОписаниеТипов("Число"));
    ТаблицаМарок.Колонки.Добавить("objnum", Новый ОписаниеТипов("Число"));
    ТаблицаМарок.Колонки.Добавить("code", Новый ОписаниеТипов("Строка", 150));
    ТаблицаМарок.Колонки.Добавить("vol", Новый ОписаниеТипов("Число", 15, 3));
    ТаблицаМарок.Колонки.Добавить("date", Новый ОписаниеТипов("Дата"));
    ТаблицаМарок.Колонки.Добавить("rvc", Новый ОписаниеТипов("Число"));
    ТаблицаМарок.Колонки.Добавить("name", Новый ОписаниеТипов("Строка", 255));
    
    // Формируем параметры запроса
    ПараметрыURL = Новый Массив;
    
    // Проверяем, загружается ли конкретный чек
    ЗагрузкаКонкретногоЧека = ЗначениеЗаполнено(ПараметрыВыборки.НомерЧека) 
                             И ПараметрыВыборки.НомерЧека <> 0;
    
    Если ЗагрузкаКонкретногоЧека Тогда
        // Загрузка марок конкретного чека
        ПараметрыURL.Добавить("cknum=" + Формат(ПараметрыВыборки.НомерЧека, "ЧГ=0"));
        
        // Опционально добавляем период для безопасности
        Если ЗначениеЗаполнено(ПараметрыВыборки.НачалоПериода) Тогда
            ПараметрыURL.Добавить("dateFrom=" + Формат(ПараметрыВыборки.НачалоПериода, "ДФ=yyyy-MM-ddTHH:mm:ss"));
        КонецЕсли;
        Если ЗначениеЗаполнено(ПараметрыВыборки.КонецПериода) Тогда
            ПараметрыURL.Добавить("dateTo=" + Формат(ПараметрыВыборки.КонецПериода, "ДФ=yyyy-MM-ddTHH:mm:ss"));
        КонецЕсли;
    Иначе
        // Массовая загрузка за период
        ПараметрыURL.Добавить("dateFrom=" + Формат(ПараметрыВыборки.НачалоПериода, "ДФ=yyyy-MM-ddTHH:mm:ss"));
        ПараметрыURL.Добавить("dateTo=" + Формат(ПараметрыВыборки.КонецПериода, "ДФ=yyyy-MM-ddTHH:mm:ss"));
    КонецЕсли;
    
    СтрокаПараметров = СтрСоединить(ПараметрыURL, "&");
    
    // Для каждой точки продаж
    Для Каждого RvcId Из ПараметрыВыборки.МассивТочекПродаж Цикл
        
        Попытка
            Endpoint = СтрШаблон("/rvc/%1/fiscaldb/marks/sold?%2", RvcId, СтрокаПараметров);
            HTTPОтвет = ОтправитьЗапросAPI("GET", Endpoint, "", Заголовки);
            
            Если HTTPОтвет.КодСостояния = 200 Тогда
                МассивМарок = ПарсироватьОтветJSON(HTTPОтвет);
                
                Если ТипЗнч(МассивМарок) = Тип("Массив") Тогда
                    Для Каждого МаркаJSON Из МассивМарок Цикл
                        СтрокаМарки = ТаблицаМарок.Добавить();
                        ЗаполнитьЗначенияСвойств(СтрокаМарки, МаркаJSON);
                    КонецЦикла;
                КонецЕсли;
            ИначеЕсли HTTPОтвет.КодСостояния = 404 Тогда
                // Нет марок для данной точки/чека - это нормально
            Иначе
                ЗаписьЖурналаРегистрации("Загрузка марок Micros",
                    УровеньЖурналаРегистрации.Ошибка,,,
                    СтрШаблон("Ошибка загрузки марок для RVC %1: HTTP %2", 
                        RvcId, HTTPОтвет.КодСостояния));
            КонецЕсли;
        Исключение
            ЗаписьЖурналаРегистрации("Загрузка марок Micros",
                УровеньЖурналаРегистрации.Ошибка,,,
                СтрШаблон("Исключение при загрузке марок для RVC %1: %2", 
                    RvcId, ОписаниеОшибки()));
        КонецПопытки;
        
    КонецЦикла;
    
    Возврат ТаблицаМарок;
    
КонецФункции
```

### 4.2 Создать функцию `ОбработатьПроданныеМарки()`

**Назначение:** Обработка таблицы проданных марок и запись в документы чеков.

**Параметры:**
- `ТаблицаМарок` - таблица с марками из API
- `ТаблицаЧеков` - таблица с загруженными чеками

**Алгоритм:**

```bsl
Функция ОбработатьПроданныеМарки(ТаблицаМарок, ТаблицаЧеков)
    
    Если ТаблицаМарок.Количество() = 0 Тогда
        Возврат;
    КонецЕсли;
    
    // ШАГ 1: Сортировка марок
    ТаблицаМарок.Сортировать("cknum Возр, ckpos Возр, date Возр");
    
    // ШАГ 2: Группировка по позициям (cknum + ckpos)
    МаркиПоПозициям = Новый Соответствие;
    
    Для Каждого СтрокаМарки Из ТаблицаМарок Цикл
        КлючПозиции = Формат(СтрокаМарки.cknum, "ЧГ=0") + "_" + 
                      Формат(СтрокаМарки.ckpos, "ЧГ=0");
        
        Если МаркиПоПозициям[КлючПозиции] = Неопределено Тогда
            МаркиПоПозициям[КлючПозиции] = Новый Массив;
        КонецЕсли;
        
        МаркиПоПозициям[КлючПозиции].Добавить(СтрокаМарки);
    КонецЦикла;
    
    // ШАГ 3: Обработка каждой позиции
    Для Каждого КлючИЗначение Из МаркиПоПозициям Цикл
        
        МаркиПозиции = КлючИЗначение.Значение;
        ПерваяМарка = МаркиПозиции[0];
        
        // Найти документ чека
        Чек = НайтиЧекПоНомеру(ПерваяМарка.cknum, ТаблицаЧеков);
        
        Если Чек = Неопределено Тогда
            ЗаписьЖурналаРегистрации("Обработка марок",
                УровеньЖурналаРегистрации.Предупреждение,,,
                СтрШаблон("Чек %1 не найден", ПерваяМарка.cknum));
            Продолжить;
        КонецЕсли;
        
        // Найти позицию в чеке по ckpos
        СтрокаТовара = Чек.Товары.Найти(ПерваяМарка.ckpos, "НомерСтроки");
        
        Если СтрокаТовара = Неопределено Тогда
            ЗаписьЖурналаРегистрации("Обработка марок",
                УровеньЖурналаРегистрации.Ошибка,,,
                СтрШаблон("В чеке %1 не найдена позиция %2", 
                    ПерваяМарка.cknum, ПерваяМарка.ckpos));
            Продолжить;
        КонецЕсли;
        
        // ШАГ 4: Добавить марки для позиции
        СуммаПоМаркам = 0;
        
        Для Каждого СтрокаМарки Из МаркиПозиции Цикл
            
            ОбъёмПродано = СтрокаМарки.vol;  // Берём напрямую из API!
            
            Если ОбъёмПродано <= 0 Тогда
                ЗаписьЖурналаРегистрации("Обработка марок",
                    УровеньЖурналаРегистрации.Предупреждение,,,
                    СтрШаблон("Марка %1: некорректный объём %2", 
                        СтрокаМарки.code, ОбъёмПродано));
                Продолжить;
            КонецЕсли;
            
            СтрокаМаркиЧека = Чек.МаркиЕГАИС.Добавить();
            СтрокаМаркиЧека.НомерПозицииЧека = СтрокаМарки.ckpos;
            СтрокаМаркиЧека.КодМарки = СтрокаМарки.code;
            СтрокаМаркиЧека.ОбъёмПродано = ОбъёмПродано;
            
            СуммаПоМаркам = СуммаПоМаркам + ОбъёмПродано;
            
        КонецЦикла;
        
        // ШАГ 5: Валидация целостности для позиции
        ОжидаемыйОбъём = СтрокаТовара.Количество * СтрокаТовара.Номенклатура.ОбъемДАЛ;
        
        Если Абс(СуммаПоМаркам - ОжидаемыйОбъём) > 0.001 Тогда
            ЗаписьЖурналаРегистрации("Валидация марок",
                УровеньЖурналаРегистрации.Ошибка,,,
                СтрШаблон("Чек %1, позиция %2: сумма марок %3л ≠ объём позиции %4л",
                    ПерваяМарка.cknum,
                    ПерваяМарка.ckpos,
                    СуммаПоМаркам,
                    ОжидаемыйОбъём));
        КонецЕсли;
        
    КонецЦикла;
    
КонецФункции
```

## 5. Ключевые моменты реализации

### 5.1 Поле `vol` - это проданный объём!

```bsl
// ПРАВИЛЬНО:
ОбъёмПродано = СтрокаМарки.vol;  // Напрямую из API

// НЕПРАВИЛЬНО:
НачальныйОстаток = ПолучитьОстатокМарки(КодМарки);
ОбъёмПродано = НачальныйОстаток - СтрокаМарки.vol;  // ❌ Неверно!
```

### 5.2 Группировка по cknum + ckpos обязательна!

```bsl
// ПРАВИЛЬНО:
КлючПозиции = Формат(cknum, "ЧГ=0") + "_" + Формат(ckpos, "ЧГ=0");
МаркиПоПозициям[КлючПозиции].Добавить(Марка);

// НЕПРАВИЛЬНО:
КлючЧека = Формат(cknum, "ЧГ=0");  // ❌ Нет ckpos!
МаркиПоЧекам[КлючЧека].Добавить(Марка);
```

### 5.3 Валидация для каждой позиции отдельно

```bsl
// ПРАВИЛЬНО:
Для Каждого Позиция Из МаркиПоПозициям Цикл
    СуммаМарок = Σ(Позиция.Марки.vol);
    ОбъёмПозиции = СтрокаТовара.Количество × Объем;
    Если СуммаМарок ≠ ОбъёмПозиции Тогда
        // Ошибка с указанием cknum и ckpos
    КонецЕсли;
КонецЦикла;

// НЕПРАВИЛЬНО:
СуммаМарокЧека = Σ(ВсеМарки.vol);  // ❌ Валидация для всего чека!
```

## 6. Тестирование

### 6.1 Тестовые сценарии

| № | Сценарий | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Продажа целой бутылки (0.7л) | 1 марка с vol=0.7 привязана к позиции |
| 2 | Продажа порции (50мл) из бутылки | 1 марка с vol=0.05 привязана к позиции |
| 3 | Продажа порции из 2 бутылок (30мл+20мл) | 2 марки (vol=0.03, vol=0.02) с одним ckpos |
| 4 | Несколько одинаковых позиций в чеке | Каждая марка привязана к своему ckpos |
| 5 | Перезагрузка конкретного чека | Загружаются только марки этого чека |

### 6.2 Критерии приёмки

- [ ] Марки группируются по cknum + ckpos
- [ ] Для каждой группы находится КОНКРЕТНАЯ строка товара по ckpos
- [ ] Поле НомерПозицииЧека заполняется значением ckpos
- [ ] Валидация выполняется для КАЖДОЙ позиции отдельно
- [ ] Ошибка валидации содержит cknum И ckpos
- [ ] Если ckpos не найден - генерируется понятная ошибка
- [ ] Поддерживается перезагрузка конкретного чека по НомерЧека

## 7. Последовательность реализации

### Этап 1: Метаданные (1 день)
1. Создать перечисления MRS_ЕдиницыИзмеренияФБ и MRS_ТипыМарокФБ
2. Добавить реквизиты в план обмена питУдаленныеКассы
3. Добавить реквизиты в справочник питМеню
4. Создать ТЧ МаркиЕГАИС в документе "Чек ККМ"

### Этап 2: Выгрузка марок (2 дня)
1. Удалить методы работы со штрихкодами
2. Создать ВыгрузитьМаркуВФискальнуюБазу()
3. Создать ОбновитьЕГАИСАтрибутыПозиции()
4. Интегрировать в существующий процесс выгрузки позиций

### Этап 3: Загрузка проданных марок (3 дня)
1. Создать ПолучитьПроданныеМаркиHTTP()
2. Создать ОбработатьПроданныеМарки() с группировкой по cknum+ckpos
3. Модифицировать ПолучениеДанныхHTTP()
4. Добавить логирование и обработку ошибок

### Этап 4: Тестирование (2 дня)
1. Протестировать все сценарии из раздела 6.1
2. Проверить валидацию объёмов
3. Протестировать перезагрузку конкретного чека
4. Проверить обработку ошибок
